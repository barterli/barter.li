Authentications [1.0]

user creation and getting user objects

create_user

creates a user if not present or returns a user with location supports two types of authentication normal email , password and outh(facebook or google)

URL

POST /api/v1/create_user

Required Parameters

  • format String

    only json supported
  • email String

    used if provider is manual
  • password String

    used if provider is manual (minimum 8 characters)
  • access_token String

    used if provider is outh(facebook or google)
  • provider String

    can be manual, facebook, google

Response Object

  • User

    The created user or existing user
